## 1. 현황 분석

### 1.1 현재 InvestHub가 가진 것

| 영역 | 기존 기능 | 성숙도 |
|------|----------|--------|
| **데이터 수집** | DART, 네이버증권, ECOS, FRED 연동 | ★★★★ |
| **AI 분석** | Gemini 기반 종합 스코어링 (기술/기본/센티먼트/리스크) | ★★★ |
| **스크리너** | 14개 필터, 프리셋 저장, KOSPI/KOSDAQ 전체 | ★★★★ |
| **내부자 거래** | DART 기반 임원 거래 모니터링 + 30분 폴링 알림 | ★★★ |
| **투자자 흐름** | 외국인/기관 순매수 추이 (네이버 스크래핑) | ★★★ |
| **매크로** | 한국(ECOS) + 글로벌(FRED) 거시경제 지표 | ★★★ |
| **테마** | 네이버 300+ 테마, 11개 카테고리 분류 | ★★★ |
| **사용자 시스템** | NextAuth 인증, 관심종목, 프리셋, 최근 조회 | ★★★ |

### 1.2 WNSTN이 가지고 있지만 InvestHub에 없는 것

| 기능 | WNSTN | InvestHub 현황 | 확장 우선순위 |
|------|-------|---------------|-------------|
| **대화형 AI 채팅** | 핵심 인터페이스 | 없음 (대시보드 중심) | **P0 (필수)** |
| **컴플라이언스 에이전트** | 자문/정보 경계 자동 관리 | 없음 | **P0 (필수)** |
| **포트폴리오 리뷰** | 보유종목 기반 성과 분석 | 관심종목만 있음 (보유량 없음) | **P1 (높음)** |
| **개인화 학습** | 대화 이력 기반 적응형 | 없음 | **P1 (높음)** |
| **교육 콘텐츠** | 투자 개념 수준별 설명 | 없음 | **P2 (중간)** |
| **이벤트 캘린더** | 실적발표/배당일 자동 알림 | 이벤트 목록은 있으나 캘린더 없음 | **P2 (중간)** |
| **VRM 대시보드** | 브로커용 고객 인텔리전스 | 없음 (B2C만) | **P3 (낮음, 향후)** |

## 2. 확장 비전

### 2.1 한 줄 포지셔닝

> "한국 주식에 특화된 규제 준수형 AI 대화 투자 어시스턴트 — 기존 데이터 분석 인프라 위에 대화형 인터페이스와 컴플라이언스를 얹는다"

### 2.2 핵심 원칙

```
1. 기존 자산 활용 극대화
   - 이미 구축된 34개 API, AI 스코어링, 데이터 파이프라인을 그대로 활용
   - 새로 만드는 것이 아니라 "대화형 껍질"을 입히는 것

2. 한국 규제 네이티브
   - WNSTN이 FINRA/SEC 기반인 반면, 처음부터 금융위/금감원 규제 기반 설계
   - 자본시장법, 금융위 AI 7대 원칙, 인공지능기본법 준수

3. 한국어 금융 전문성
   - 한국어 금융 용어, DART 공시체계, KRX 거래 관행에 최적화
   - PER/PBR뿐 아니라 공매도, 대주거래, 공시 용어 등 한국 특화 개념

4. 점진적 확장
   - Phase 1 (대화형 AI) → Phase 2 (개인화) → Phase 3 (B2B 플랫폼)
```

### 2.3 WNSTN과의 차별점

| 항목 | WNSTN | InvestHub 확장안 |
|------|-------|-----------------|
| **시장** | 글로벌 90+ 거래소 | 한국 주식 전문 (KOSPI/KOSDAQ) |
| **데이터 깊이** | 넓고 얕은 글로벌 커버리지 | 깊은 한국 시장 커버리지 (DART 공시, 내부자 거래, 기관/외국인 흐름, 300+ 테마) |
| **규제** | FINRA/SEC 기반 → 한국 적응 중 | 한국 금융 규제 네이티브 |
| **AI 모델** | 자체 Bespoke Financial AI | Gemini 2.5 Flash + 한국 금융 특화 프롬프트 |
| **배포** | SaaS (해외 서버) | 국내 배포 (Vercel icn1 서울) |
| **가격 데이터** | 유료 데이터 피드 의존 | 네이버증권 + DART (무료 공공 데이터 활용) |

### 3.2 에이전트별 상세

#### 투자정보 에이전트 (Engagement Agent)

사용자와 직접 대화하며 기존 InvestHub 데이터를 자연어로 전달하는 프론트 에이전트.

| 기능 | 활용할 기존 API | 새로 구현할 것 |
|------|---------------|--------------|
| **종목 분석** | `/api/stocks/[ticker]`, `/api/ai-score/[ticker]` | AI 스코어를 자연어로 해설하는 프롬프트 |
| **시장 동향** | `/api/market`, `/api/ranking`, `/api/macro` | 시장 상황을 스토리텔링으로 요약 |
| **포트폴리오 리뷰** | `/api/stocks/[ticker]/investor`, AI 스코어 | 사용자 보유종목 일괄 분석 + 성과 비교 |
| **공시 해석** | `/api/events`, `/api/insider`, `/api/block-holdings` | DART 공시를 평이한 한국어로 번역 |
| **테마 분석** | `/api/themes`, `/api/themes/[no]` | 테마 트렌드를 이야기 형태로 설명 |
| **배당 분석** | `/api/dividends` | 배당 투자 관점 정보 제공 |
| **교육** | 없음 | 한국 주식 투자 용어/개념 교육 DB 구축 |

#### 컴플라이언스 에이전트 (Compliance Agent)

모든 AI 응답이 한국 금융 규제를 준수하도록 실시간 검증하는 게이트키퍼.

```
사용자 질문 입력
       │
       ▼
┌──────────────────────┐
│  의도 분류 (Intent)   │
│                       │
│  ├─ 정보 요청         │──→ "삼성전자 PER은?" → 데이터 응답
│  ├─ 분석 요청         │──→ "삼성전자 재무 어때?" → 팩트 기반 분석
│  ├─ 교육 요청         │──→ "PBR이 뭐야?" → 교육 콘텐츠
│  ├─ 비교 요청         │──→ "삼성 vs SK하이닉스" → 비교 데이터
│  └─ 자문 요청 ⚠️     │──→ "삼성전자 지금 사?" → 차단 + 교육 전환
│                       │
│  한국 규제 체크:       │
│  ├─ 자본시장법 준수    │  (투자권유 금지)
│  ├─ 금융위 AI 7대원칙  │  (투명성, 공정성, 안전성 등)
│  ├─ 인공지능기본법     │  (고위험 AI 분류 대응)
│  └─ 개인정보보호법     │  (투자 데이터 처리 동의)
│                       │
│  필수 면책 문구:       │
│  └─ "본 정보는 투자   │
│     판단의 참고용이며  │
│     투자 권유가 아닙   │
│     니다."            │
│                       │
│  기록:                │
│  └─ 전체 대화 로그    │──→ DB 저장 (감사 추적용)
└──────────────────────┘
```

**한국 특화 차단 규칙:**

| 차단 대상 | 예시 | 전환 응답 |
|-----------|------|----------|
| 매매 추천 | "삼성전자 사야 해?" | "삼성전자의 최근 실적과 AI 분석 점수를 안내해 드리겠습니다" |
| 목표가 제시 | "카카오 목표가 알려줘" | "증권사 컨센서스 목표가와 현재 밸류에이션을 비교해 드리겠습니다" |
| 포트폴리오 구성 | "1000만원으로 뭘 사?" | "현재 시장 상위 종목과 섹터별 밸류에이션을 안내해 드리겠습니다" |
| 타이밍 조언 | "지금이 저점이야?" | "최근 기술적 지표(RSI, MACD)와 거래량 추이를 보여드리겠습니다" |

#### 분석 에이전트 (Analysis Agent)

기존 AI 스코어링 엔진을 강화하여 대화 맥락에 맞는 분석을 수행.

| 기능 | 기존 기반 | 확장 내용 |
|------|----------|----------|
| **종합 스코어** | `scoring.ts` (Gemini 기반) | 대화 맥락에 맞는 요약 수준 조절 |
| **기술적 분석** | RSI, MACD, SMA, 볼린저밴드 등 | 자연어 해설 추가 ("과매수 구간 진입") |
| **기본적 분석** | PER/PBR/EPS/배당률 | 동종업계 비교 자동화 |
| **뉴스 센티먼트** | 네이버/구글 뉴스 크롤링 | 실시간 뉴스 요약 + 감성 변화 추이 |
| **리스크 평가** | 변동성, ATR 기반 | 관심종목 간 상관관계 분석 추가 |

#### 핵심 구현: 오케스트레이터

```typescript
// lib/chat/orchestrator.ts 개념 설계

type Intent =
  | 'stock_analysis'     // "삼성전자 분석해줘"
  | 'market_overview'    // "오늘 시장 어때?"
  | 'portfolio_review'   // "내 관심종목 리뷰"
  | 'disclosure_query'   // "최근 공시 알려줘"
  | 'theme_analysis'     // "2차전지 테마 어때?"
  | 'macro_query'        // "금리 전망은?"
  | 'education'          // "PER이 뭐야?"
  | 'comparison'         // "삼성전자 vs SK하이닉스"
  | 'insider_query'      // "내부자 거래 있어?"
  | 'dividend_query'     // "배당 정보 알려줘"
  | 'investment_advice'  // ⚠️ 차단 대상

async function orchestrate(userMessage: string, context: ChatContext) {
  // 1. 의도 분류 (Gemini로 분류하거나 규칙 기반)
  const intent = await classifyIntent(userMessage)

  // 2. 컴플라이언스 체크
  if (intent === 'investment_advice') {
    return complianceAgent.redirectToEducation(userMessage, context)
  }

  // 3. 필요한 데이터 수집 (기존 API 활용)
  const data = await dataFetcher.fetchForIntent(intent, userMessage, context)

  // 4. 응답 생성 (Gemini + 한국 금융 특화 프롬프트)
  const response = await engagementAgent.generate(intent, data, context)

  // 5. 면책 문구 추가
  return addDisclaimer(response)
}
```

#### DB 스키마 확장

```prisma
model ChatSession {
  id        String        @id @default(cuid())
  userId    String
  user      User          @relation(fields: [userId], references: [id])
  title     String?       // 자동 생성된 세션 제목
  messages  ChatMessage[]
  createdAt DateTime      @default(now())
  updatedAt DateTime      @updatedAt

  @@index([userId, updatedAt])
}

model ChatMessage {
  id            String      @id @default(cuid())
  sessionId     String
  session       ChatSession @relation(fields: [sessionId], references: [id])
  role          String      // 'user' | 'assistant' | 'system'
  content       String      @db.Text
  intent        String?     // 분류된 의도
  blocked       Boolean     @default(false)  // 컴플라이언스 차단 여부
  dataSources   String?     // 사용된 데이터 소스 목록 (감사용)
  createdAt     DateTime    @default(now())

  @@index([sessionId, createdAt])
}
```

## 11. 결론

InvestHub는 이미 **한국 주식 투자정보의 데이터 레이어**를 탄탄하게 구축해 놓았다. 34개 API 엔드포인트, DART/네이버/ECOS/FRED 연동, Gemini 기반 AI 스코어링이 모두 동작하고 있다.

WNSTN 모델의 확장은 **이 데이터 레이어 위에 대화형 인터페이스라는 새로운 접점을 추가**하는 것이다. 핵심은:

1. **새로 만드는 것을 최소화** — 기존 34개 API를 그대로 활용하되, 자연어로 접근 가능하게 함
2. **한국 규제를 처음부터 내장** — WNSTN이 FINRA→한국 적응에 고전하는 부분을 네이티브로 해결
3. **한국 주식 데이터의 깊이** — DART 공시, 내부자 거래, 외국인/기관 수급 등 WNSTN이 제공하지 못하는 한국 특화 데이터

WNSTN이 "넓고 얕은 글로벌 커버리지"라면, InvestHub 확장안은 **"좁고 깊은 한국 주식 전문 AI 어시스턴트"**로 차별화한다.
